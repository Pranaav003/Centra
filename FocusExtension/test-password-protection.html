🔒 Focus Extension Password Protection Test

📋 Test Instructions

  1. Load the Extension: Go to chrome://extensions/ and load the Focus extension from this folder
  2. Enable Password Protection: Go to the web app at http://localhost:3001 and enable password protection in Settings → Privacy
  3. Add Some Websites: Add a few websites to the block list (e.g., youtube.com, facebook.com)
  4. Test Web App: Try to remove a website from the web app - you should be prompted for a password
  5. Test Extension: Try to remove a website from the extension popup - you should be prompted for the same password

✅ Expected Behavior

  • Web App: Password modal shows: "Enter your password to remove [site] from block list"
  • Extension: Password modal shows: "Enter password to remove [site] from block list"
  • Both: Use the same password as set in Settings → Privacy
  • Both: Incorrect password shows error message
  • Both: Cancel button aborts the removal

🔧 Implementation Details

Web App Changes:

  • Added pendingSiteToRemove state variable
  • Modified removeBlockedSite() to check password protection
  • Created performSiteRemoval() for actual removal logic
  • Updated password modal to show site-specific message
  • Enhanced password handlers to process both blocking toggle and site removal

Extension Changes:

  • Added pendingSiteToRemove variable
  • Created removeSite() and performSiteRemoval() functions
  • Updated delete button onclick to use removeSite()
  • Enhanced password submit handler for both actions
  • Added dynamic password message display

🎯 Test Cases

  1. Password Disabled: Site removal should work immediately without password prompt
  2. Password Enabled - Correct Password: Site removal should proceed after entering correct password
  3. Password Enabled - Wrong Password: Should show error and not remove site
  4. Password Enabled - Cancel: Should abort removal and not remove site
  5. Consistency: Same password works for both web app and extension
